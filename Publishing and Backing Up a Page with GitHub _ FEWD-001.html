Unit 1 /Lesson 1 /Assignment 4

Publishing and Backing Up a Page with GitHub

Estimated Time: 1-2 hours

In the previous assignment you ported your “About me” page to a standard web page project structure, using separate HTML and CSS files. You were able to preview your “About me” page in the browser, but we’ve not yet published it on the web (or, “deployed” it) so that others could type in a URL and view it.

In this assignment we’ll take our first steps with Git and GitHub in order to back up and publish the “About me” page from the previous assignment. We'll learn a lot more about Git and GitHub later in this unit, but for now, it's enough to know that Git is a widely used version control system (and version control is about backing up and managing changes to code). It's a piece of software that lives on your computer. GitHub, in contrast, is a cloud-based platform for storing and collaborating on projects using Git.

We're going to install GitHub Desktop, which is a GUI (graphical user interface) that will allow us to easily backup and publish our work. Later in this unit, after we've learned how to use the command line, you’ll learn to control Git/GitHub with commands as well.

At the end of this assignment, we've included a screencast that walks through adding Git to a project, backing up to GitHub, and publishing a static webpage using GitHub pages. We recommend you both read the content and watch the screencast.

Be aware that newcomers to Git and GitHub often find them somewhat difficult to first learn. The reason we're introducing them now, near the very beginning of your journey through this course is twofold. First, you've just learned how to create a basic webpage, but you don't have a way to share that work with the outside world, and in this assignment you'll learn just enough Git/GitHub to publish a web page. Second, complex topics like version control are easier to learn incrementally, so it's a good idea to learn a little now and build on this understanding later.

If you get stuck on this assignment, work through it with your mentor at your next session or reach out for help on Slack.

Sign up for GitHub and Install a GitHub client

  1. GitHub accounts are free to create. Go to the sign up link and follow the instructions. When you get to the plan selection screen, be sure to select the free option.
  2. Once you've signed up for GitHub, it's time to download a GUI client. Download the installer here and the install the app.
  3. Open GitHub Desktop and input the username and password from step 1.

Create a new Git Repo for "About me" and publish using GitHub Pages

A Git repo (short for repository) is a project that's been initialized with Git. We need to take our existing "About me" project folder and turn it into a repo, which will allow us to back it up on GitHub, and then publish our page using GitHub Pages.

Follow these steps:

  1. To add "About me" as a repo, click the plus icon on the top left of GitHub desktop, then select the "Add a repo" option. Inside the file selection interface, navigate to your "About me" project folder and add it. You'll be warned that this folder is not a repo, and asked to create a repo, which you should do.
  2. Now we're ready to commit our files. For now, it's enough to know that when you commit files, you're adding your work to Git's history of your project -- in other words, it's a lot like saving a document in Word or Google docs. To commit your files, click on your About me project in the left hand sidebar of GitHub Desktop. In the bottom middle of the interface, there's a place to add a commit description and summary. For the description you can write something like "V1 of about me page" and you can put a lengthier description in the description input.
  3. Click "Commit to master", and then click the "Publish" button in the top right corner of the interface to backup your work to GitHub.com.
  4. Take a moment to view your work on GitHub. You can do this by right clicking on the repo in the left hand side of GitHub Desktop, then selecting the "View on GitHub" option. You should see that there’s a complete version of your folder, including every file, that you can click around and review on GitHub.
  5. Next we're going to publish the page to the web using GitHub pages, a tool GitHub provides that lets you host simple websites. To do this, we need to create a new branch. We'll discuss branching in depth in a later lesson; for now, all you need to know is that "a branch is a different version of your code stored within the same repo. We're going to name our new branch gh-pages, which is a special branch name in GitHub that you can use to publish to the web.
  6. Click the add a branch button at the top of GitHub Desktop. It looks like this: . Enter gh-pages as your branch name, then click "Create branch". Your new gh-pages branch will be identical to the master branch.
  7. Now it's time to publish your gh-pages branch. Click the "Publish" button in the top right corner of GitHub Desktop.
  8. To get the link for your live page, go to your repo on GitHub.com (you can use the same method as you did in step 4, and then click the "Settings tab". Scroll down until you see the GitHub pages section. Click on the link that's provided there to view your live page.

Again, if you got stuck on any of these steps, don't worry. Your mentor can get you on the right path at your next session. We’ll continue to provide specific directions whenever you need to use Git/GitHub until we discuss it in more depth and practice it more later this unit.
